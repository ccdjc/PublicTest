# 探测器状态管理系统

## 功能概述

这是一个用于管理32个探测器状态的实时监控系统，具备司机室占用状态检测和智能屏蔽功能。

## 核心功能

### 1. 探测器状态管理
- 支持32个探测器（编号1-32）
- 四种状态：正常、离线、报警、故障
- 每秒实时更新和输出状态

### 2. 司机室占用检测
- 通过输入数组第二位字节识别司机室占用状态
- 支持三种状态：无占用(0)、一端占用(1)、二端占用(2)

### 3. 智能屏蔽逻辑
- **一端司机室占用**：自动屏蔽1号探测器的报警状态
- **二端司机室占用**：自动屏蔽17号探测器的报警状态
- 只屏蔽报警状态，其他状态（正常、离线、故障）不受影响

## 数据格式

### 输入数组格式
- 最小长度：2字节
- 完整格式：34字节（2字节头部 + 32字节探测器状态）
- 第1字节：其他数据（可自定义）
- 第2字节：司机室占用状态（0=无占用，1=一端占用，2=二端占用）
- 第3-34字节：32个探测器的状态数据

### 探测器状态值
- 0：正常
- 1：离线
- 2：报警
- 3：故障

## 运行示例

程序运行时会显示：
1. 当前司机室占用状态
2. 32个探测器的实时状态
3. 屏蔽信息（当有探测器被屏蔽时）

### 屏蔽示例
```
屏蔽信息：1号探测器报警已被屏蔽（一端司机室占用）
探测器01: 正常 (原状态: 报警 - 已屏蔽)
```

## 核心算法

### 屏蔽逻辑
```c
switch (cabin_status) {
    case CABIN_END1:  // 一端司机室占用
        if (detector[0] == ALARM) {
            output[0] = NORMAL;  // 屏蔽1号探测器报警
        }
        break;
    case CABIN_END2:  // 二端司机室占用
        if (detector[16] == ALARM) {
            output[16] = NORMAL;  // 屏蔽17号探测器报警
        }
        break;
}
```

## 实际应用集成

在实际应用中，需要：
1. 替换 `generate_test_data()` 函数为实际的数据接收接口
2. 修改输出函数以适配实际的输出需求
3. 根据需要调整屏蔽逻辑和探测器编号
